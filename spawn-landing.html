01
HOW IT
WORKS

Five steps. One direction. No setup ritual.

AUTH
CONNECT ONCE

Auth with GitHub and your AI agents. Credentials live in a private ~/.spawn repo that syncs across machines. Runs exactly once.

spawn auth github login
CLONE
POINT AT A TASK

Paste any GitHub issue, PR, or branch URL. Spawn creates an isolated Docker container with deps from the shared cache. Takes under 90 seconds.

spawn clone github.com/org/repo/issues/42
WORK
AGENT TAKES OVER

In --auto mode Claude Code or Codex reads the issue, writes code, runs tests. You do other things. In --ssh mode you enter the container and work interactively.

spawn clone ... --auto
SHIP
PR OPENS ITSELF

Agent commits, pushes, opens a pull request. Session log uploads to a private Gist and a link lands in the PR comment. Full audit trail.

→ github.com/org/repo/pull/88
DIE
ENVIRONMENT SELF-DESTRUCTS

Container is destroyed. No leftover state. No drift. No "works on my machine." Next task gets a fresh spawn. That's the whole point.

Container terminated. ✓

03
FEATURES

Built for how AI agents actually work. Not for how you think they should work.

PARALLEL ISOLATION

Run issue-42 and issue-43 simultaneously. Separate containers, separate agents, zero interference. Share nothing except the package cache.

spawn swarm 42 43 44
SHARED CREDENTIALS

One auth.json for all containers. No token rotation issues. Each project keeps its own session state while sharing the login.

SPAWN_SHARE_AUTH=1
TWO AI AGENTS

Claude Code and Codex pre-installed in every environment. AGENTS.md auto-loaded with issue context. They read the task. They know what to do.

Claude Code + Codex
SESSION BACKUP

On every push, AI session logs auto-upload to a private GitHub Gist. A link is posted to the PR comment. Full audit trail of what the agent did.

Auto-gist on git push
CLOUD STATE

Your ~/.spawn directory syncs to a private GitHub repo on auth. Switch machines. Restore everything. Never reconfigure. One command, same state.

GitHub-backed state repo
PLAYWRIGHT MCP

Optional Chromium sidecar. Claude Code gets full browser automation through MCP protocol. Agents that can see pages, click buttons, fill forms.

--mcp-playwright
04
THE API

The whole interface.
Fits in your head.

$ spawn auth github login
Connect GitHub. Creates ~/.spawn state repo. Runs once.
$ spawn auth claude login
Authorize Claude Code. Shared across all containers.
$ spawn clone <url>
Spawn an isolated container for any GitHub issue, PR, or branch.
$ spawn clone <url> --auto
Spawn, let the agent work, open PR, die. Zero babysitting.
$ spawn clone <url> --ssh
Spawn and enter. Work interactively inside the container.
$ spawn open <url>
Re-enter an existing environment. Runs up + tmux attach.
$ spawn list
All live containers. What's running, what's done.
$ spawn mcp-playwright <url>
Add Chromium sidecar for agents that need to browse.
